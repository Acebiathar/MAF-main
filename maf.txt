# 1. Main Idea of the System

**Medicine Availability Finder**

A web platform where:

* Patients search for medicines
* Pharmacies update available drugs
* The system shows **which pharmacy has the drug, the price, and alternatives**
* Users receive **notifications when drugs are available or out of stock**

---

# 2. Types of Users (Roles)

Your system should have **3 user roles**:

### 1️⃣ Patients (Users)

They can:

* Search medicines
* View pharmacies with the medicine
* View drug prices
* View **alternative medicines**
* Receive **notifications**
* Save favorite pharmacies

---

### 2️⃣ Pharmacists

They can:

* Register their pharmacy
* Add medicines
* Update stock
* Set prices
* Mark drugs **in stock / out of stock**
* Respond to medicine requests

---

### 3️⃣ Admin

The admin manages everything.

Admin controls:

* Approve pharmacies
* Manage users
* Manage medicines database
* View system statistics
* Remove fake pharmacies
* Monitor drug availability

---
